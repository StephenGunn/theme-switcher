## What Makes Each App Match

### Neovim
- Uses official colorscheme plugins
- Bufferline: Solid tabs, transparent editor
- Icons: Match buffer state (selected/unselected)
- Lualine: Auto-detects theme

### Ghostty
- Built-in theme support
- Themes: "Gruvbox Dark", "Catppuccin Mocha", "Nord"

### Hyprland
- Color config files in each theme directory
- Variables: $background, $foreground, $color0-15, etc.

### GTK Apps
- Native GTK themes from AUR
- Updates all three configs (GTK2/3/4)
- Includes cursor and icon themes

### Qt Apps (Dolphin, Kate, etc.)
- Kvantum theme engine
- Environment: `QT_STYLE_OVERRIDE=kvantum`
- Config: `~/.config/Kvantum/kvantum.kvconfig`

### Waypaper
- GTK app â†’ Uses GTK theme
- Plus: Custom CSS generated from pywal
- Result: Perfect color matching

### Rofi
- Native .rasi theme files
- Each theme has matching rofi theme

### Hyprpanel
- Native JSON themes
- Located in each theme directory

### Starship / Dunst
- Generated by pywal
- Use pywal color templates

## Troubleshooting

### Qt Apps (Dolphin) Don't Change

**Solution:**
1. Check Kvantum is installed: `pacman -Qi kvantum`
2. Check env var: `echo $QT_STYLE_OVERRIDE` (should be "kvantum")
3. Restart the Qt app completely
4. If still not working, run: `kvantummanager` and select theme manually

### GTK Apps Don't Change

**Solution:**
1. Check GTK theme installed: `ls /usr/share/themes/ | grep -i gruvbox`
2. Restart the GTK app
3. Check config: `cat ~/.config/gtk-3.0/settings.ini`

### Neovim Bufferline Still Transparent

**Solution:**
- Restart Neovim completely
- Check: `:highlight BufferLineFill` should show a background color
